# Editorial Guidelines

**Target Audience**: Senior professionals, CTOs, CFOs, and technical leads in Finance and AI/ML.
**Voice**: Authoritative, analytical, forward-looking. "CFO meets CTO".

## Core Principles

1. **Bottom Line Up Front (BLUF)**:
    - Start every article with a strict "Executive Summary" (3-4 bullets).
    - Give the value immediately. Don't bury the lead.

2. **No Fluff / No Hype**:
    - Avoid adjectives like "game-changing", "revolutionary", "unprecedented" unless backed by data.
    - Ban words: "Delve", "In conclusion", "As an AI language model", "Landscape".
    - Use concrete metrics: "Reduces audit time by 40%" instead of "Improves efficiency".

3. **Technical Depth**:
    - Don't just explain *what* (e.g., "Transformers are good").
    - Explain *how* and *why* (e.g., "Attention mechanisms allow processing of non-sequential financial ledgers...").
    - Always include code snippets (Python/SQL) or architectural diagrams (Mermaid) where relevant.

4. **Strategic Context**:
    - Always answer: "What is the ROI?"
    - Discuss implementation challenges, risks, and compliance (GDPR/EU AI Act).

## Formatting

- **Headings**: Clear, descriptive. No "Introduction" or "Conclusion".
- **Lists**: Use short, punchy bullets.
- **Emphasis**: use **bold** for key concepts, *italics* for emphasis.
- **Links**: Cite reputable sources (arXiv, official docs, regulatory bodies).

## Structure Checklist

- [ ] **Title**: Specific benefit-driven (e.g., "Implementing RAG for GAAP Compliance").
- [ ] **Executive Summary**: The "TL;DR" for busy executives.
- [ ] **The Problem**: Current state friction.
- [ ] **The Solution**: Technical/Architectural deep dive.
- [ ] **The Implementation**: Code/Diagrams.
- [ ] **The Impact**: Business value / ROI.
